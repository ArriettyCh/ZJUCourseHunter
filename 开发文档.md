# ZJU Course Hunter 开发文档

## 1. 项目概述

### 1.1 项目目标

为浙大学生提供一个**开箱即用**的自动抢课工具。用户下载后直接运行，在真实教务界面中登录，并挑选想抢的课程，确认后程序自动循环抢课直到全部选中。

### 1.2 核心设计理念

| 原则 | 说明 |
|------|------|
| **零配置** | 用户不需要懂技术，不需要手动复制 Cookie 或课程编码 |
| **所见即所得** | 在教务系统真实界面中选课，而非输入晦涩的课程代号 |
| **可控浏览器** | 使用 Playwright 托管浏览器，而非依赖用户本地浏览器环境 |
| **轻量** | 项目小而精，不做过度设计 |

---

## 2. 用户使用流程

```
$ python main.py

[自动] 检测环境，安装 Playwright 浏览器（首次运行）
                 ↓
[自动] 弹出浏览器窗口。如果首次登录，打开教务网登录页，用户登录，并自动记录下用户的账号密码。否则采用 requests 自动登录
                 ↓
[自动] 向页面注入"购物车"UI（浮动面板 + 添加按钮）
                 ↓
[用户] 浏览课程，点击"加入抢课列表"，逐门添加
                 ↓
[用户] 确认列表无误，点击"开始抢课"
                 ↓
[自动] 浏览器关闭，提取 Cookie，进入抢课循环
                 ↓
[自动] 对每门课每秒尝试一次选课请求
                 ↓
[自动] 全部成功后退出，或 Cookie 过期时重新登录（优先用 requests 自动登录，失败时回退到浏览器登录）
```

## 3. 技术架构

### 3.1 技术栈

| 组件 | 选型 | 理由 |
|------|------|------|
| 语言 | Python 3.12 | 生态好，学生群体熟悉 |
| 浏览器自动化 | Playwright | 比 Selenium 更现代，自带浏览器管理，不依赖用户环境 |
| HTTP 请求 | requests | 抢课循环阶段用，轻量快速 |
| 打包分发 | PyInstaller 或 pip | 两种都支持 |

### 3.2 模块划分

```
zju-grab/
├── main.py              # 入口：环境检测 → 启动流程
├── env_setup.py         # 环境检测与自动安装
├── auth.py              # 登录流程（Playwright 打开浏览器，等待用户登录）
├── course_picker.py     # 注入购物车 UI，收集用户选课列表
├── grabber.py           # 抢课循环（requests 发请求）
├── inject.js            # 注入到教务系统页面的前端 JS（购物车逻辑）
└── requirements.txt     # playwright, requests
```
